# 分流规则

**分流规则**决定流量走直连、代理还是 REJECT。与 `模块/` 里的 URL 重写（去开屏）不是同一类东西。

## 你需要关注的文件

| 文件 | 类型 | 说明 |
|------|------|------|
| `局域网.yaml` | 分流 | 局域网直连 |
| `国内直连.yaml` | 分流 | **合并去重** — Direct+微信+B站+苹果中国+国内域名/IP/ASN |
| `去广告.yaml` | 分流 | **合并去重** — Repcz Reject + Sukka 广告域名集 |
| `国外/*.yaml` | 分流 | **按服务分开** — AI/Telegram/流媒体/游戏/兜底等，对应不同节点策略 |

## 不用管的目录

| 目录 | 说明 |
|------|------|
| `_upstream/` | CI 临时下载的上游镜像，仅用于合并脚本，**不提交到 main** |

## 在 Egern.yaml 中的引用

```
分流/局域网.yaml      → DIRECT
分流/国内直连.yaml    → 国内直连
分流/去广告.yaml      → REJECT
分流/国外/AI.yaml     → ChatGPT
分流/国外/Telegram.yaml → Telegram
…（其余国外规则各对应自己的策略组）
```

每日 **11:00 北京时间** GitHub Actions 从上游拉取 → 合并去重 → 同步到 `main`。
